# CoachOS — MVP Scope

## Принцип приоритизации

Использован фреймворк MoSCoW. Критерии:
- **Must-have**: без этого продукт не имеет смысла или нельзя защитить диплом
- **Should-have**: значительно улучшает опыт, реализуемо в срок
- **Could-have**: nice-to-have, не критично
- **Won't-have (now)**: откладываем, слишком сложно или не нужно для MVP

## Won't Have (Now) — Future Enterprise

| Фича | Причина откладывания |
|---|---|
| PDF-экспорт отчётов | Требует Puppeteer или gofpdf, отдельный эпик |
| Tactical Board (Konva.js) | Canvas-рисование — отдельный инструмент |
| Video Analysis | Storage, обработка видео — другой продукт |
| Wearables (GPS/HR) | Hardware интеграция — MVP слишком рано |
| Push-уведомления (FCM) | Требует мобильное приложение |
| Real OpenAI/Claude AI | Затраты токенов, непредсказуемые ответы на защите |
| Billing (Stripe) | Post-PMF |
| Real-time чат (WebSocket) | Отдельный эпик, пока хватит уведомлений |
| Mobile app (React Native) | После web-MVP |
| Multi-tenancy | После набора 10+ клубов |

## Почему именно этот scope реалистичен

### Технические оценки
- **Backend**: ~75 endpoints, 23 таблицы БД, ~3500 строк Go кода → ~3 недели senior fulltime
- **Frontend**: ~35 страниц, ~60 компонентов, ~5000 строк TSX → ~3 недели senior fulltime
- **Итого**: 6–8 недель в режиме fulltime, или 12–16 недель при 4 часах в день (дипломный режим)

### Архитектурные решения под скоуп
- GORM вместо sqlc → на 23 таблицах экономит 2–3 дня на написание репозиториев
- Zustand вместо Redux → на frontend экономит 1–2 дня
- Mock AI → нет зависимости от внешнего API, работает всегда на защите
- In-app notifications вместо push → нет mobile app dependency
- On-screen reports вместо PDF → нет Puppeteer dependency

### Что останется впечатляющим
- Player Development Index — инновационная фича, которой нет у конкурентов
- Smart Training Load — защита от детских травм, убедительный аргумент
- Coach Command Center — профессиональный UI как у SaaS-продуктов
- Parent Transparency Mode — эмоциональная фича для продажи
- AI Assistant (mock) — выглядит как продуктовая инновация

## Definition of Done для MVP

- [ ] Все Must-have функции реализованы и работают end-to-end
- [ ] Docker Compose поднимает весь стек одной командой
- [ ] Seed-данные создают демо-академию с реалистичными данными
- [ ] Все роли работают с защищёнными маршрутами
- [ ] Frontend адаптивен на mobile/tablet/desktop
- [ ] Нет критических ошибок при демо-сценарии
- [ ] Задокументированы API и архитектура
- [ ] Написаны ключевые тесты (auth, RBAC, PDI расчёт)
