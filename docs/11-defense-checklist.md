## 4. Код и архитектура

- [ ] Структура backend соответствует Clean Architecture
  - `domain/` — чистые модели, ноль зависимостей
  - `service/` — бизнес-логика, нет HTTP/SQL
  - `handler/` — только HTTP
  - `repository/` — интерфейсы + postgres-реализация
- [ ] Структура frontend соответствует Feature-Sliced Design
  - `app/ → pages/ → widgets/ → features/ → entities/ → shared/`
  - Нет запрещённых импортов (соседних/верхних слоёв)
- [ ] `go build ./cmd/api` собирается без ошибок
- [ ] `npm run build` во frontend собирается без TypeScript-ошибок
- [ ] `make lint` проходит (или есть понимание, какие предупреждения можно объяснить)
- [ ] Все TODO / FIXME либо закрыты, либо сознательно отложены в roadmap

## 8. Безопасность и соответствие требованиям

- [ ] JWT access token хранится в памяти (Zustand), не в `localStorage`
- [ ] Refresh token — httpOnly cookie
- [ ] Пароли захешированы bcrypt
- [ ] CORS настроен whitelist
- [ ] Rate limiting включён
- [ ] RBAC middleware на всех защищённых endpoint
- [ ] Club-scoping: тренер видит только свой клуб
- [ ] Медицинские заметки видны только coach/admin
- [ ] Parent видит только своего ребёнка

## 9. Риски и план Б

| Риск | Вероятность | План Б |
|------|-------------|--------|
| Backend не поднимается на защите | Низкая | Скриншоты дашборда, видео-скринкаст |

| Вопрос про нагрузку/масштабирование | Средняя | "Monolith-first, microservice-ready, roadmap Phase 2–4" |
