# ADR-001: Go + Fiber для Backend

**Статус**: Accepted  
**Дата**: 2024-01-01

## Контекст

Нужно выбрать технологию для backend REST API платформы CoachOS.

## Варианты

1. Go + Fiber
2. Node.js + Fastify
3. Python + FastAPI

## Решение

**Go + Fiber v2**

## Обоснование

- **Производительность**: Go компилируемый, Fiber на fasthttp (~10x быстрее net/http) — важно при аналитических запросах
- **Типизация**: строгая система типов исключает ошибки на этапе компиляции
- **Concurrency**: goroutines идеальны для асинхронных уведомлений
- **Малый footprint**: бинарник ~15MB, контейнер ~20MB
- **Fiber API**: похож на Express.js, низкий порог входа

## Последствия

- Нет автоматической OpenAPI документации (нужен отдельный инструмент)
- Меньше готовых библиотек чем в Node.js экосистеме
- Требует Go 1.22+
